Restaurant Roulette

Live Demo Repository

Everyone has been a part of a conversation like this one at some point in their life. For me, this conversation happens almost daily, followed by what feels like an eternity scrolling on Yelp.

Today, people have the luxury of choice. Whether it be the type of car they choose to drive, or what brand canned-corn they want to buy for Thanksgiving dinner, the options are endless.

The result is a 'first-world-problem' of being paralyzed by choice, or not being able to easily make a decision. With around 5,000 restaurants listed on Yelp in San Francisco, choosing a place to eat is no easy feat.

Restaurant Roulette was created in the hopes of removing one less decision out of a person's day.

The web app takes in user criteria such as price point, distance, and number of results, and uses these parameters to return a list of random nearby restaurants.

The user's location is retrieved using the browser's built-in Geolocations API and combined with the user's choices in order to create an endpoint for Google's Map API and Places Library. JavaScript is used to pass the endpoint to Google Maps API and the Places Library, and is also used to parse through JSON data, and return a randomized array of data.
